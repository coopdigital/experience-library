Guidelines

You should aim to have more rows than columns in your table.

This is because:

  • wide tables can be difficult for people to read and understand
  • the fewer columns your table has, the more likely it is that the table will fit onto the screen of a smaller device

If your table needs to be wider than the smallest device that is supported:

  • allow the user the scroll the table left and right
  • provide cues that there is more content that people can scroll to, for example the table could be formatted so the table looks cut off on the screen
  • research the implementation with real people to make sure it’s as accessible as possible

You should use tables to present data only.

You should not use images in tables.

The heading for a table is called a caption. Your caption should be clear and descriptive of what users will find in the table.

Table markup for accessibility

Tables must be made accessible by using appropriate markup.

Captions

A caption is the heading for a table. It appears above the table as a heading.

Most screen readers announce the content of captions.

Headers and footers

Both columns and rows of a table can have headings these indicate the groups of data within the table.

If there are a lot of rows in your table, you can use table footers which appear as headers but at the bottom of the table to help people remember the data that the table contains if they have scrolled down the page a long way.

Row headings

Headings can be used to indicate grouped data along a row.

Top left cell

The top-left cell of a table is often blank as there is no overarching header for the row headings and the content of the table has been described by the caption.

This cell can be left blank.
