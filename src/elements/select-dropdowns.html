Guidelines

We do not advise using select dropdowns because research shows that some users find them difficult to use. We follow the same guidelines as gov.uk.

Whenever possible try to ask questions with fewer options and use radio buttons instead.

Go to the guidelines for radio buttons.

Watch a video about how some users struggle with select dropdowns.

Sometimes due to technical constraints, you may need to use a select dropdown.

All attributes of the select dropdown are designed using values from the foundations. Do not change the styling of the select dropdown or the associated styles.

Go to the foundations.

Select field

Do not change the styling of the select field.

We’ve designed the select dropdown using values from the foundations. By default, the form elements have no margin on the bottom. When using more than one form element, add a margin-bottom of 24px. This will space form fields and labels evenly down the page.

Go to the foundations.

Labels

Select dropdowns must have a label associated with them. The label must be short and make it clear what information you’re asking for.

Error messages

Use error messages with the validation component.

Give error messages after the user submits the form. Do not give them as someone is entering information into each input. This can cause stress and frustration.

Error messages should appear above the relevant input. This means it is read out by assistive technology in an order that makes sense.

Focus ring

The focus ring appears as a purple outline around any object, including select dropdowns, when someone selects it.

Do not remove the focus ring. People who navigate with a keyboard cannot know where they are on the screen without it. The focus ring lets them know which input they have selected.
