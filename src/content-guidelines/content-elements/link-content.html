Link content

A link (short for hyperlink) is an HTML object that allows the user to go to a new location when they click or tap it.

  • View in Figma
  • Go to the design and engineering guidelines on links

Guidance

Link text must:

  • tell the user where it’s taking them
  • be action orientated, for example, ‘Choose an option’, ‘Find more information about the discount’, ‘Discover…’
  • be underlined
  • tell the user if they’re leaving our site, for example, ‘For more information visit citizensadvice.org.uk’
  • use a full url if it contains no symbols, it is short and has a maximum of one /, for example, bbc.co.uk/sport. If it’s a complex url, use a word or words to link, for example ‘See the Co-op blog post on how to have a more sustainable Christmas’.

Links should usually be set to open in the same tab. See the Accessibility section of this page for more information on why.

Sometimes you might have a good reason for wanting to set a link to open in a new tab. For example, if opening it in the same tab could cause a user to lose their progress through a form.

If you set a link to open in a new tab, you must state within the link text that the link opens in a new tab, for example, Find out more about Co-op Membership (opens in a new tab).

Accessibility

Do not use text such as ‘click here’, or anything that does not make sense when read out of context — screen readers can read link text in isolation without the surrounding content.

We usually set links to open in the same tab because setting links to open in new tabs can cause accessibility issues. It takes control away from the user and might make it difficult for them to navigate back to our site.

Links do not need to include a title tag. This accessibility feature is not used any more as it’s not that helpful for visually impaired people.

For more information see our general guide on accessibility
