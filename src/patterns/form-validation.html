Guidelines

Use form validation when a person has given information that you cannot use. Then show them an error message that helps them to fix the problem.

Build forms that are flexible enough to allow people to give data in a way that they would naturally. For example, allow people to enter a postcode in upper or lower case, with or without a space.

Trigger form validation when information:

  • is missing
  • does not match the expected format

How form validation works

Only trigger form validation when the user submits the form – not as they are still typing. Doing this can frustrate people if, for example, they type slowly.

When you trigger form validation:

  • re-load the page after it’s been submitted with the error summary at the top of the page
  • move the keyboard focus to the error summary
  • list the errors in bullet points in the error summary
  • make each bullet point an anchor link that, when selected, takes the user to the relevant field
  • keep the information that was submitted in the form fields so the user can see their error
