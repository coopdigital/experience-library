Content guidelines

Use form validation when a person has given information that you cannot use. Then show them an error message that helps them to fix the problem.

Activate form validation when information:

  • is missing
  • does not match the expected format

Allow people to give data in a way that they would naturally. For example allow people to enter:

  • a postcode in upper or lower case, with or without a space
  • their full name in one field, to allow for names that do not fit into a first name and surname structure

Writing validation messages

Validation messages must:

  • be specific about what the user needs to do fix the error, for example, ‘enter the amount of hours worked in whole numbers’, not ‘hours entered invalid’
  • start with an action, for example, ‘enter’ or ‘select’
  • be written in plain English
  • make sense when read out of context, for example, ‘enter your full name’, not ‘fill in the field’

The form validation pattern is made up of an:

  • error summary at the top of the page
  • error message above the field

The errors listed in bullet points in the error summary start with a lower case letter.

The error message above the field start with a capital letter.

Both do not end with a full stop.

Design guidelines

Validation happens when the user submits the page or form – not as they are still typing. Validating before the user has submitted can frustrate people, for example, if they type slowly.

When you use validation it means that:

  • the page reloads after it’s been submitted, with the error summary at the top of the page
  • ‘Error: ’ is added to the beginning of the page title in the code – this means it will be read out by screen readers when the page reloads
  • the keyboard focus moves to the error summary
  • the errors are listed in bullet points in the error summary
  • each bullet point goes to an anchor link that, when selected, takes the user to the relevant field
  • the incorrect information that was submitted stays within the field so the user can see their error
  • the error messages are repeated above the relevant fields
